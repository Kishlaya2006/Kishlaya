# Mini Tutorial: Git, GitHub and Markdown
## What is Git
Git is a version control system used to track changes in files. It allows you to save versions of your code, view history, and revert to previous versions if needed.
------------------------------------------------------------------------------------------------------------------
## How to Push Code to GitHub
1. Initialize a Git repository:
```
git init
```
2. Add files to staging:
```
git add .
```
3. Commit the changes:
```
git commit -m "Initial commit"
```
4. Add the remote repository:
```
git remote add origin <repository-url>
```
5. Push the code to GitHub:
```
git push -u origin main
```
-----------------------------------------------------------------------------------------------------------------

## What is Markdown

Markdown is a lightweight syntax used to format text. It is commonly used in README files and documentation.

-----------------------------------------------------------------------------------------------------------------

## Basic Markdown Syntax

### Headings

```
# Heading 1
## Heading 2
### Heading 3
```

### Lists

Unordered list:

```
- Item 1
- Item 2
- Item 3
```

Ordered list:

```
1. First
2. Second
3. Third
```

### Code

Inline code:

```
`code`
```

Code block:

```
print("Hello")
```

### Links

```
[Link Text](https://example.com)
```

### Bold and Italic

```
**Bold**
*Italic*
```
